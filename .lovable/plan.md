

# خطة بناء موقع Gnsader لسيرفر SAMP

## ملخص المشروع
موقع بلون ذهبي لسيرفر SA-MP يحتوي على صفحات (الرئيسية، القوانين، الوظائف، العقوبات) مع لوحة تحكم للمطورين محمية بكود 106، والبيانات مخزنة في قاعدة بيانات Supabase.

## التصميم
- **اللون الرئيسي**: ذهبي (#D4AF37) مع خلفية داكنة (#1a1a2e)
- **القائمة**: ثلاث شخطات (hamburger menu) تفتح sidebar بالأقسام
- **الاتجاه**: RTL (عربي بالكامل)

## هيكل قاعدة البيانات (Supabase)

4 جداول:
1. **settings** — رابط الديسكورد ورابط تحميل اللعبة (key/value)
2. **rules** — القوانين (id, title, content, order)
3. **jobs** — الوظائف (id, title, description, order)
4. **sanctions** — العقوبات مع أقسام (id, category, title, description, order)

## الصفحات والمكونات

### 1. Layout مشترك
- **Navbar**: شعار "Gnsader" + أيقونة hamburger menu
- **Sidebar**: قائمة جانبية تظهر عند الضغط (الرئيسية، القوانين، الوظائف، العقوبات)

### 2. الصفحة الرئيسية (`/`)
- شعار وعنوان السيرفر
- زرين: "دخول الديسكورد" و "تحميل اللعبة" (الروابط من قاعدة البيانات)

### 3. صفحة القوانين (`/rules`)
- عرض القوانين من الداتابيس بشكل accordion أو قائمة

### 4. صفحة الوظائف (`/jobs`)
- عرض الوظائف بكروت

### 5. صفحة العقوبات (`/sanctions`)
- عرض العقوبات مصنفة حسب الأقسام

### 6. لوحة المطورين (`/admin`)
- شاشة إدخال الكود (106) — بدون تسجيل دخول حقيقي
- بعد إدخال الكود الصحيح:
  - تعديل رابط الديسكورد ورابط التحميل
  - إدارة القوانين (إضافة/تعديل/حذف)
  - إدارة الوظائف (إضافة/تعديل/حذف)
  - إدارة العقوبات وأقسامها (إضافة/تعديل/حذف)

## ملاحظة أمنية
الكود 106 هو حماية بسيطة على مستوى الواجهة فقط. جداول Supabase ستكون بدون RLS لتسهيل الإدارة حالياً. يمكن تعزيز الأمان لاحقاً.

## خطوات التنفيذ
1. إعداد Supabase Cloud وإنشاء الجداول الأربعة مع بيانات أولية
2. تحديث الثيم للون الذهبي مع خلفية داكنة
3. بناء Layout مع hamburger menu وsidebar
4. بناء الصفحات الأربع (رئيسية، قوانين، وظائف، عقوبات)
5. بناء لوحة المطورين مع شاشة الكود وأدوات الإدارة
